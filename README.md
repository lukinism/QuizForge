# QuizForge

> Веб-платформа для создания, выдачи, прохождения, проверки и анализа онлайн-тестов.

[![Python](https://img.shields.io/badge/Python-3.14-blue.svg)](https://www.python.org/)
[![FastAPI](https://img.shields.io/badge/FastAPI-0.136-009688.svg)](https://fastapi.tiangolo.com/)
[![MongoDB](https://img.shields.io/badge/MongoDB-8.0-47A248.svg)](https://www.mongodb.com/)

QuizForge — это приложение для проведения тестирований с ролями, группами, выдачей тестов, ручной проверкой, импортом JSON, загрузкой медиафайлов и PDF-отчетами.

Проект подходит для учебных групп, внутренних аттестаций, тренировочных тестов и сценариев, где нужны не только вопросы с вариантами ответа, но и развернутые ответы, практические задания и проверка проверяющим.

## Возможности

- Роли `admin`, `examiner`, `student`
- Авторизация через JWT и HttpOnly cookies
- Создание, редактирование, публикация и архивирование тестов
- Импорт тестов из JSON
- Публичные тесты и приватные ссылки
- Группы тестируемых
- Ссылки для вступления в группы
- История вступлений в группу
- Удаление и блокировка участников группы
- Отзыв ссылок вступления
- Выдача тестов из вкладки группы
- Мониторинг прохождения теста в реальном времени
- Досрочное завершение теста для всей группы или отдельного тестируемого
- Режим “все вопросы сразу”
- Режим “по одному вопросу”
- Возможность вернуться к предыдущему вопросу
- Возможность пропускать вопросы
- Инструкция перед прохождением теста
- Автоматическая проверка объективных вопросов
- Ручная проверка развернутых ответов и практических заданий
- Комментарии проверяющего
- Отправка ответа на доработку
- Подтверждение итогового балла
- Личный профиль тестируемого
- История результатов
- PDF-отчеты по пользователю, группе и тесту
- Загрузка изображений, аудио, видео и файлов
- Защищенная обработка загружаемых файлов

## Типы вопросов

Поддерживаемые типы:

- `single_choice` — один правильный вариант
- `multiple_choice` — несколько правильных вариантов
- `text_answer` — краткий текстовый ответ
- `free_answer` — развернутый ответ с ручной проверкой
- `matching` — соответствие
- `ordering` — сортировка по порядку
- `fill_blank` — заполнение пропусков
- `image` — вопрос с изображением
- `audio` — вопрос с аудио
- `video` — вопрос с видео
- `file` — вопрос с файлом
- `code` — вопрос с фрагментом кода
- `practical` — практическое задание с ручной проверкой

## Роли

### Администратор

Администратор может:

- управлять пользователями;
- менять роли;
- блокировать и разблокировать пользователей;
- видеть все тесты;
- видеть все попытки и результаты;
- создавать и редактировать любые тесты;
- формировать PDF-отчеты.

### Проверяющий

Проверяющий может:

- создавать тесты;
- редактировать свои тесты;
- импортировать тесты из JSON;
- создавать группы;
- создавать ссылки для вступления в группы;
- видеть, кто вступил в группу и когда;
- удалять и блокировать участников;
- выдавать тесты группе;
- наблюдать прохождение теста в реальном времени;
- завершать тест досрочно;
- проверять развернутые ответы и практические задания;
- отправлять ответы на доработку;
- подтверждать итоговый балл;
- формировать отчеты по своим тестам и группам.

### Тестируемый

Тестируемый может:

- зарегистрироваться и войти в систему;
- вступить в группу по ссылке;
- проходить публичные тесты;
- проходить приватные тесты;
- проходить тесты, выданные группе;
- дорабатывать ответы, если проверяющий вернул работу;
- смотреть свои результаты и комментарии проверяющего.

## Быстрый запуск

Создайте `.env`:

```bash
cp .env.example .env
```

Создайте виртуальное окружение:

```bash
python3.14 -m venv .venv
source .venv/bin/activate
```

Для Windows PowerShell:

```powershell
py -3.14 -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Установите зависимости:

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Запустите MongoDB 8 локально и убедитесь, что она доступна по адресу из `MONGO_DSN`.

Стартуйте приложение:

```bash
python -m uvicorn app.main:app --reload
```

Откройте приложение:

```text
http://localhost:8000
```

По умолчанию используется MongoDB:

```text
mongodb://127.0.0.1:27017
```

## Настройка

Настройки берутся из `.env`.

| Переменная | Описание | Значение по умолчанию |
| --- | --- | --- |
| `APP_NAME` | Название приложения | `QuizForge` |
| `DEBUG` | Режим отладки | `true` |
| `SECRET_KEY` | Секретный ключ для JWT | `change-me` |
| `JWT_ALGORITHM` | Алгоритм JWT | `HS256` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Время жизни access-токена | `30` |
| `REFRESH_TOKEN_EXPIRE_DAYS` | Время жизни refresh-токена | `7` |
| `MONGO_DSN` | Строка подключения к MongoDB | `mongodb://127.0.0.1:27017` |
| `MONGO_DB` | Название базы данных | `testing_system` |
| `ACCESS_COOKIE_NAME` | Название access-cookie | `access_token` |
| `REFRESH_COOKIE_NAME` | Название refresh-cookie | `refresh_token` |
| `REPORT_STORAGE_DIR` | Папка PDF-отчетов | `storage/reports` |
| `UPLOAD_STORAGE_DIR` | Папка загруженных файлов | `storage/uploads` |

Для публичного или боевого окружения обязательно замените `SECRET_KEY`.

## Создание первого администратора

После запуска MongoDB выполните:

```bash
python -m app.scripts.create_admin \
  --email admin@example.com \
  --username admin \
  --password StrongPassword123 \
  --full-name "System Admin"
```

Скрипт:

- создаст нового администратора;
- или повысит существующего пользователя до роли `admin`.

## Ручная проверка

Ручная проверка включается автоматически для:

- `free_answer`
- `practical`

Логика:

1. Тестируемый завершает тест.
2. Обычные вопросы проверяются автоматически.
3. Развернутые ответы и практические задания получают статус `Требует проверки`.
4. У проверяющего появляется предупреждение в панели.
5. Проверяющий открывает попытку.
6. Проверяющий ставит баллы и пишет комментарий.
7. Проверяющий может отправить работу на доработку.
8. При доработке тестируемый видит только вопросы, которые требуют ручной проверки.
9. После повторной отправки попытка снова уходит на проверку.
10. Проверяющий подтверждает итоговый балл.

## PDF-отчеты

Приложение умеет формировать PDF-отчеты:

- по попытке пользователя;
- по группе;
- по тесту.

PDF генерируются средствами приложения и сохраняются в папку, указанную в `REPORT_STORAGE_DIR`.

## Заметки для разработки

- В проекте используется легкий внутренний ODM в `app/core/odm.py`.
- Модели MongoDB инициализируются при старте FastAPI-приложения.
- PDF-отчеты создаются через Pillow.
- В репозитории есть демонстрационные JSON-файлы для быстрого тестирования.

## Чеклист перед production

- Заменить `SECRET_KEY`
- Выключить `DEBUG`
- Настроить HTTPS
- Защитить MongoDB
- Настроить secure cookies
- Добавить CSRF-защиту для форм
- Добавить антивирусную проверку загружаемых файлов
- Не коммитить `.env`, `storage/`, `reports/`, дампы базы и локальные логи
